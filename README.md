# targil0
# מערכת מידע לניהול עיצוב וייצור (D&M System)
**מגישות:** שרי אדלר וצוות הסטודנטיות (שלישייה)  
**קורס:** בניית מערכות מידע - שלב 1  
**נושא הפרויקט:** עיצוב וייצור: דגמים, חומרי גלם, מוצרים, ספקים ועובדים.

---

## 1. מבוא ותיאור המערכת
מערכת זו תוכננה לנהל את מחזור החיים המלא של מפעל ייצור ועיצוב. המערכת עוקבת אחר תהליך הפיתוח החל משלב ה**דגם (Design)** הכולל מפרטים טכניים מורכבים, דרך ניהול **שרשרת האספקה** (ספקים וחומרי גלם), ועד ל**ייצור הפיזי** בקווי ייצור וניהול **כוח האדם** והמשמרות.

### גבולות המערכת:
* **אגף מחקר ופיתוח (R&D):** ניהול דגמים ומפרטי JSON.
* **אגף לוגיסטיקה:** מעקב אחר מלאי חומרי גלם והזמנות רכש מספקים.
* **אגף ייצור:** ניהול קווי ייצור ומוצרים מוגמרים.
* **אגף משאבי אנוש:** שיבוץ עובדים למחלקות וניהול משמרות.

---

## 2. תכנון לוגי - דיאגרמת ERD
המערכת מורכבת מ-**9 ישויות מרכזיות**. לכל ישות הוגדרו לפחות 5 תכונות (Attributes) כדי להבטיח פירוט נתונים מרבי ודיוק בתהליכי העבודה.

## 3. נירמול ותלויות פונקציונליות
כל הטבלאות תוכננו לעמוד ברמת נירמול **BCNF** כדי למנוע כפילויות נתונים ולשמור על שלמות המידע. המכריע (Determinant) בכל תלות הוא מפתח מועמד.

| שם הישות | מפתח ראשי (PK) | תלויות פונקציונליות (FD) | רמת נירמול |
| :--- | :--- | :--- | :--- |
| **Design (דגם)** | `D_id` | `D_id -> D_name, D_description, D_date, JSON_Specs` | BCNF |
| **Product (מוצר)** | `P_id` | `P_id -> P_name, P_price, P_weight, P_date` | BCNF |
| **RawMaterial (חומר)** | `R_id` | `R_id -> R_name, R_price, Stock_Quantity, Unit_Measure` | BCNF |
| **Supplier (ספק)** | `S_id` | `S_id -> Company_Name, Phone, Address, Supplier_Metadata` | BCNF |
| **Employee (עובד)** | `E_id` | `E_id -> E_name, E_familyName, Role, E_date` | BCNF |
| **Department (מחלקה)** | `DE_id` | `DE_id -> DE_name, Location, Budget, Manager_Name` | BCNF |
| **Product_Line (קו)** | `PL_id` | `PL_id -> Capacity, Status, Factory_Location, Last_Maintenance` | BCNF |
| **SupplyOrder (הזמנה)** | `Order_id` | `Order_id -> Order_date, Order_status, Total, Shipping_Method` | BCNF |
| **Work_Shift (משמרת)** | `WS_id` | `WS_id -> WS_type, WS_time, WS_date, WS_notes` | BCNF |

## 4. מימוש פיזי ובסיס נתונים
בסיס הנתונים מומש ב-**PostgreSQL**. 
* **שימוש ב-JSON:** השדות `JSON_Specs` ו-`Supplier_Metadata` הוגדרו כסוג `JSONB` לאחסון מידע גמיש.
* **תאריכים משמעותיים:** המערכת כוללת שדות תאריך כמו `P_date` (ייצור) ו-`Order_date` (הזמנה) לצורך שאילתות.

---

## 5. מילוי נתונים (Data Population)
הוזנו מעל 500 רשומות בטבלאות המרכזיות באמצעות:
1. **קוד פייתון:** שימוש בספריית `Faker` לנתונים ריאליסטיים.
2. **אתרים חיצוניים:** Mockaroo ליצירת רשימות עובדים.
3. **קובץ CSV:** הזנת נתוני מחלקות.
